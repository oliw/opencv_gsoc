Markerless AR Sample For Android

This is a sample application for Markerless Augmented Reality using OpenCVForAndroid. It uses extended code by Khvedchenia Ievgen (see Mastering OpenCV book)

Using the Camera Calibration

For Augmented Reality to work, you need to provide your device camera's intristic parameters so that the graphics rendered on top of the camera feed are warped to ensure a realistic effect. To retrieve these parameters you need to calibrate your camera. The main activity won't begin unless the camera calibration has happened.

You will need to print off the 11x4 OpenCV asymmetric circles' grid from here . Then, open the Camera Calibration screen and point the camera over the printed pattern. When the pattern is found, tap the screen to collect the image. Collect several photos and then press the Calibrate button. Once calibration has successfully completed you can return to the main screen.

You can change this by importing your own patterns in the YML format from the application menu.

Using your own patterns

To use your own patterns you must use the 'Import Patterns' from the main menu and select a directory containing your patterns in YML format.

The YML file expects the following attributes:

  • width - width of pattern
  • height - height of pattern
  • keypoints - the serialized vector of OpenCV keypoints detected in the pattern
